05 / CONFIGURATION

Một config, nhiều dòng gem.

Cấp tối đa mặc định là 5. Mỗi gem family có thể ghi đè giới hạn và ánh xạ từng level sang MMOItems item id riêng.

GLOBAL DEFAULTLevel 5 out of the box PER-GEM OVERRIDEAny family can set its own cap TEMPLATE MAPResolve every next-level item
config.ymlYAML
default-max-level: 5

gems:
  satthuong:
    max-level: 5
    type: GEM_STONE
    levels:
      1: SATTHUONG_1
      2: SATTHUONG_2
      3: SATTHUONG_3
FLEXIBLE BY DEFAULT

Global default

default-max-level áp dụng cho mọi family chưa có giới hạn riêng.

Per-gem override

max-level trong family luôn được ưu tiên hơn giá trị global.

Mapped + fallback

Map template rõ ràng hoặc bật convention fallback theo GEMID_LEVEL.

plugins/OmniGemLevel/config.yml
default-max-level: 5

resolver:
  convention-fallback: true
  id-case: upper

gems:
  satthuong:
    max-level: 5
    type: GEM_STONE
    levels:
      1: SATTHUONG_1
      2: SATTHUONG_2
      3: SATTHUONG_3
      4: SATTHUONG_4
      5: SATTHUONG_5

07 / FAQ

Câu hỏi thường gặp.

Các tình huống nên kiểm tra trước khi đưa gem progression lên server chính.

Thiếu template level tiếp theo thì sao?

Plugin không tiêu hao gem trên chuột và không sửa item. Kiểm tra config levels và MMOItems item id.

Nâng cấp có tính success-rate không?

Không. Exact-pair upgrade thành công khi template level tiếp theo tồn tại và hợp lệ.

Cùng id nhưng khác level có bị đè không?

Không. Trường hợp đó được trả về MMOItems để dùng socket hợp lệ tiếp theo nếu có.

Socket color và giới hạn item còn hoạt động không?

Có. Non-upgrade case dùng luồng MMOItems gốc; upgrade case áp dụng gem mới bằng API MMOItems.

Không tìm thấy phần phù hợp.
